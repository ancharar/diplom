# Scientific Project Manager (ScienceFlow)

Веб-приложение для управления проектами и задачами научных групп: ведение проектов и задач, оформление литературных источников по ГОСТу, поиск научных статей на arXiv, формирование отчётности, учёт публикаций и уведомления.

## Пример использования

### 1. Вход в систему

Откройте приложение в браузере и войдите по email и паролю. Если аккаунта ещё нет — перейдите по ссылке «Регистрация».

### 6. Литература проекта

Раздел литературы хранит источники проекта со счётчиками по типам (статьи, книги, источники с arXiv). Источники можно искать, фильтровать по типу и добавлять вручную, а также находить научные статьи прямо на arXiv и добавлять их в проект одним кликом. Для каждого источника формируется оформление по ГОСТу с возможностью копирования.

Конструктор позволяет настраивать шаблоны библиографических ссылок: выбирать тип источника, добавлять поля (авторы, название, год и т.д.) и разделители, сохранять и редактировать существующие шаблоны.

## Инструкция по запуску


#### Backend

1. Установите и запустите PostgreSQL и MongoDB локально либо поднимите только их через Docker:
   ```bash
   docker compose up -d postgres mongodb
   ```

2. Создайте базу PostgreSQL (если СУБД установлена локально):
   ```sql
   CREATE DATABASE scientific_pm;
   ```

3. Создайте и активируйте виртуальное окружение:

   Windows (PowerShell):
   ```powershell
   python -m venv venv
   .\venv\Scripts\Activate.ps1
   ```
   Linux / macOS:
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   ```

4. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

5. Создайте файл `.env` из шаблона и проверьте настройки подключения к БД (для локального запуска `DB_HOST` и `MONGO_HOST` должны быть `localhost`):
   ```bash
   cp .env.example .env
   ```

6. Примените миграции и создайте индексы MongoDB:
   ```bash
   python manage.py migrate
   python manage.py create_mongo_indexes
   ```

7. Создайте суперпользователя:
   ```bash
   python manage.py createsuperuser
   ```

8. (Опционально) заполните БД тестовыми данными:
   ```bash
   python manage.py seed_data
   ```

9. Запустите сервер разработки:
   ```bash
   python manage.py runserver
   ```
   Backend: **http://localhost:8000**

#### Frontend

1. Перейдите в каталог фронтенда:
   ```bash
   cd frontend
   ```

2. Установите зависимости:
   ```bash
   npm install
   ```

3. (Опционально) настройте адрес API. По умолчанию используется `http://localhost:8000/api/v1`. Чтобы переопределить — создайте `frontend/.env`:
   ```
   VITE_API_URL=http://localhost:8000/api/v1
   ```

4. Запустите dev-сервер:
   ```bash
   npm run dev
   ```
   Frontend: **http://localhost:5173**

5. Сборка для продакшена:
   ```bash
   npm run build      # результат в frontend/dist
   npm run preview    # локальный просмотр собранной версии
   ```
